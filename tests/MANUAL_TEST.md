# 端到端人工测试

## 1. 现代控制台与目标自动规划

1. 运行 `scripts\start.ps1`，浏览器应打开本地 Focus Buddy 页面，不出现旧 Tk 主窗口。
2. 检查桌面宽屏和缩窄窗口：布局应保持完整；切换“减少动效”后，背景漂浮、卡片入场和小猫呼吸动画停止。
3. 输入 `45分钟完成高数作业第三章`，点击“帮我安排场景”。
4. 应出现课程入口、解题工具、资料参考等场景化选项，而不是 JSON 或连续打字框。
5. 取消一项，并用“应用 / 页面 / 进程”选择框和输入框补充一个工具。
6. 点击“开始专注”，不应出现“这份配置需要澄清”的阻断提示。

## 2. 网址白名单与浏览器桥接

1. 不加载浏览器桥接，添加 `bilibili.com` 网站域名白名单并点击开始；应明确提示先连接桥接，不得静默启动或扣币。
2. 在 Edge 的 `edge://extensions` 打开开发人员模式，加载项目的 `browser_extension` 文件夹，刷新 Focus Buddy；“网址识别桥接”卡片应变绿并显示当前域名。
3. 添加 `bilibili.com` 后开始，分别打开 `https://www.bilibili.com/` 和其子域名；超过宽限时间仍不应提醒或扣币。
4. 切换到未允许的其他域名，超过宽限后才提醒并扣除本轮清醒值；保持在同一已允许域名不得循环误扣。
5. 开始后停留在标题为 Focus Buddy 的 `127.0.0.1` 控制台；它应显示“自动放行”，不得因为运行在 Edge 中而提醒。
6. 关闭或停用扩展并等待 6 秒以上，状态应显示桥接断开，不得把最后一个域名误用于之后的浏览器窗口。
7. 在已经开始的域名白名单会话中让桥接心跳短暂断开；界面应显示“网址识别暂时断开 · 本轮不扣分”，不得产生新提醒或扣币。
8. 保持“页面关键词”默认选项，直接粘贴完整网址并加入；标签应自动显示为“网址 · 域名”，而非页面标题规则。
9. 检查 `.runtime`：不得记录完整 URL 或浏览历史；桥接只在内存短暂保留活动域名和标题。

## 3. 未规划直接开始

1. 只输入 `完成今天的编程任务并跑通测试`，不要先安排场景。
2. 直接点击“开始专注”。
3. 程序应自动补入 VS Code、终端等推荐项并启动，不显示技术错误。

## 4. VS Code 智能关联推荐

1. 建立一个只允许 `Code.exe` 的白名单会话。
2. 保持 VS Code 前台，再打开文件资源管理器。
3. 前 4 秒不应出现越界吐槽；连续使用后应出现小猫关联推荐，而不是真正越界警告。
4. 选择“本轮允许”，本轮再次打开资源管理器不再询问。
5. 新一轮选择“以后都允许”，重启后关联仍生效；成长页的“我教会它的关系”数量增加。
6. 对浏览器进行同类测试：标题含 `GitHub`、`documentation` 或课程关键词才应推荐；普通视频或娱乐页不应因“浏览器”身份自动获准。

## 5. 真正越界提醒与小猫动效

1. 先点击页面右上角“叫 Luna 出来”，无需开始会话即可预览。
2. 小猫与对白应出现在屏幕中下方，而不是固定右下角；当前选中的小猫从右侧连续滑入并横向摊在页面下方，尾巴和四肢全程完整可见。
3. 小猫应具有稳定的脸型、身体、毛色和透明边缘；运动全程不应变脸、变形、叠帧、露出紫色背景或突然跳帧。点击“走神·摇头拒绝”“失败·推翻茶杯”“成功·腮红害羞”，应分别看到局部摇头、杯子倾倒、双颊变红，结束后能复位。
4. 对白使用深色悬浮气泡，并明确显示“清醒值、预计少得猫币”；预览模式必须声明不实际扣除。
5. 连续预览三次，目标依次模拟视频、热搜和购物页面；文案应命中对应类别，第三次出现短促的左右“盯人”动作，按钮仍可正常点击。
6. 在白名单会话切到无关联应用并超过宽限，真实提醒应显示目标名称、剩余时间和本轮损失。
7. 弹窗不会遮满屏幕、连续轰炸或使用人格侮辱；达到最大提醒次数后仍继续计时，但不再反复弹窗。

## 6. 黑名单与完全离屏

1. 黑名单模式加入 `哔哩哔哩` 和 `微博` 标题关键词，打开相应页面超过宽限时间，应提醒。
2. 完全离屏模式启动后停止键鼠操作，不应提醒；持续操作超过宽限时间后应提醒。

## 7. 暂停、结束与长期成长

1. 打开“成长记录”，把小猫改名为“饭团”；侧栏、预览按钮、桌面提醒和刷新后的页面都应保留新名字。
2. 依次选择“奶牛警长”和“云朵布偶”，侧栏、专注页、成长房间与“叫它出来”的桌面弹窗必须同步换猫；刷新页面后仍保留，名字“饭团”和累计成长不变。
3. 暂停会话，确认倒计时不减少且不触发提醒；继续后恢复。
4. 完成一轮，经验、金币、累计分钟、猫粮和历史记录增加；无提醒时“干净完成”计数增加。
5. 触发一次偏航后，清醒值应从 100 降至 92，并明确显示预计少 2 猫币；连续偏航扣分递增，但不低于 20。
6. 检查成长房间的五阶段时间线：阶段 0–1 显示对应毛色的幼猫，阶段 2 起切换到成猫；从幼猫、认主、少年、陪伴到守护，显示体型应逐级增长而非只有文字变化。
7. 累计完成 60 分钟后，小猫应从“刚到家的幼猫”成长为“会认主的小猫”；成长条重新指向下一阶段，页面和桌面提醒都应同步更新素材与大小。
8. 提前结束另一轮，本轮不增加小猫成长分钟、经验和金币，也不倒扣已有成果；放弃次数增加，并播放短促的生气动作。
9. 正常完成一轮后播放害羞冒爱心动作；完成达到条件的场次后，成长页应显示相应徽章、七日柱状图和下一枚徽章提示。

## 8. AI 规划与 Ollama 离线降级

1. 启动 Ollama 并确认已有 `qwen3.5:9b`，保持“本机 AI 增强”开启。
2. 输入 `50分钟完成Python数据分析项目，并用Excel核对结果`；状态应经历“AI 正在理解”，结果来源显示本机 AI，并推荐代码编辑器、终端、文件资源管理器和表格软件。
3. 输入 `25分钟修改简历，禁止刷B站和微博`；应规划为黑名单模式，禁止项包含哔哩哔哩和微博，不应把娱乐网址放入白名单。
4. 关闭“本机 AI 增强”后重复输入，页面应立即使用本地场景库，不能调用 Ollama。
5. 重新开启 AI，再关闭 Ollama；输入目标后应明确显示本地回退原因，不能卡死界面或阻止开始专注。
6. 给模型制造超时或无法解析的输出时，系统仍应回退到本地场景库；监测、关联软件判断、计时、奖励记录和本地提醒继续正常工作。

## 9. 目标场景库与本地记忆

1. 依次输入 Python 开发、实验报告、答辩 PPT、Excel 报表、Obsidian 笔记、视频剪辑和腾讯会议目标；应分别出现对应工作软件，项目文件夹只出现在需要素材或文件管理的任务中。
2. 对“45分钟完成Python课程作业”删除 VS Code、手动加入 PyCharm 和文件资源管理器后开始，再结束本轮。
3. 输入“继续完成Python课程作业”，应优先复用刚确认的 PyCharm 组合；`.runtime/goal_scenario_memory.json` 只记录目标摘要和白名单，不含页面正文。
4. 关闭 AI 开关后连续点击不同目标的“帮我安排场景”，界面应即时响应，不等待 Ollama。

## 10. 自定义宠物领养

1. 在“成长记录”选择一张 PNG、JPG 或 WebP 宠物照片，输入名字并点击“生成并领养”；生成期间按钮应显示处理状态，页面不得出现 `Failed to fetch`。
2. 生成后应出现四段本地成长素材，当前宠物立即切换为新伙伴；侧栏、专注舞台、成长房间和桌面提醒均使用该宠物。
3. 分别预览害羞、摇头和推杯动作，自定义宠物应复用同一套流畅动画，不回退成内置橘猫。
4. 刷新页面后，自定义伙伴仍在；切换到内置小猫再切回来，成长分钟和名字不丢失。
5. 删除未选中的自定义宠物只移除该宠物；删除当前宠物后安全回退到“橘子汽水”，本机对应原图与成长图目录同时删除。
6. 断开网络后重复上传与生成，功能仍应正常；照片只存在 `.runtime/custom_pets`（安装版为 `%LOCALAPPDATA%\FocusBuddyAI\custom_pets`）。

## 11. 白噪音花园与猫咪剧情舞台

1. 打开“声音花园”，应能按雨幕、溪流、海岸、鸟鸣、晴日等类别筛选；页面不再展示本地路径、文件格式限制或“LOCAL / LOOP”统计标签，NCM 加密文件不得出现在曲目列表。
2. 选择任一声音，底部悬浮播放器应显示曲名与作者，播放键、上一首、下一首、进度条和音量均可操作，页面不得卡顿或横向溢出。
3. 刷新页面，上次选择的声音和音量应保留；音频仍从本机读取，不产生外网请求。长音频应使用 HTTP 206 分段读取，而非一次性加载整个文件。
4. 回到“本轮专注”：未开始显示放松小猫，专注中显示电脑前工作的小猫，刚偏航时显示困惑动作，确认偏航后切换为打翻杯子并带水滴动效。
5. 正常完成时应切换跳舞庆祝并出现彩纸；达到成长节点时显示礼物盒场景；提前结束显示受惊场景。每个场景的短句应与动作一致。
6. 打开“减少动效”后，镜头缩放、晃动、彩纸和声音视觉波形应停止或显著减弱，但功能与文本保持可用。

## 统一检查

- 源码版任务管理器中的解释器来自 `D:\Agent\focus-supervisor-agent-ai\.venv`；安装版只显示 `FocusBuddyAI.exe`。
- Web 服务只监听 `127.0.0.1`，端口位于 8765–8775；页面能正常加载安全响应头。
- 没有截图、按键内容或完整浏览历史落盘。
- 域名规则由新鲜的浏览器桥接心跳匹配；桥接断开时不会退化成窗口标题猜测。
- 关联软件先经过软宽限与用户确认，未知软件仍按原越界规则处理。
- 原生小猫提醒与网页成长数据在关闭并重启应用后仍可正常工作。
